About Our Project

Project Overview

The 42 Webserv project is a complete HTTP server implementation developed in C++ as part of the 42 School curriculum. Our server handles multiple client connections simultaneously, processes HTTP requests, and serves appropriate responses while strictly adhering to RFC standards.

Project Components

The server is composed of several key components working together:

🔌 Socket Management

Implements non-blocking socket operations using poll() for handling multiple connections efficiently. Includes IP address parsing, port binding, and connection queuing.

📝 Configuration Parser

Reads and validates server configuration files with support for multiple servers, routes, error pages, and CGI settings. Handles both JSON and custom configuration formats.

📨 Request Parser

Parses incoming HTTP requests including headers, body, and chunked transfer encoding. Validates request syntax according to RFC 7230 standards.

📤 Response Generator

Constructs proper HTTP responses with status codes, headers, and body content. Handles both static file serving and dynamic content generation.

⚙️ CGI Handler

Implements CGI 1.1 specification with support for multiple interpreters (Python, PHP, Perl). Manages environment variables, standard I/O streams, and timeout control.

🧪 Testing Framework

Comprehensive test suite including unit tests, integration tests, and performance benchmarks. Automated testing with custom client implementations.

Technical Specifications

  • Language: C++98 (with some C++11 features where allowed)
  • Concurrency: Single-process, event-driven architecture
  • Protocols: HTTP/1.1 (RFC 7230-7235)
  • Supported Methods: GET, POST, DELETE, HEAD
  • Configuration: Multiple servers with different host:port combinations
  • Performance: Handles 1000+ concurrent connections on modern hardware
